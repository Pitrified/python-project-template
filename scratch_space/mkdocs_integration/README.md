# MkDocs Integration

Integrate MkDocs with Material theme for professional project documentation, including auto-generated API docs.

---

## Overview

### Selected Approach: Root-Level `mkdocs.yml`

Place `mkdocs.yml` at the repository root with `docs/` as the source directory.

**Why this approach:**

- Standard convention; most MkDocs projects use this layout.
- Easier CI/CD integration (GitHub Actions expects root config by default).
- Clear separation: config at root, content in `docs/`.
- Acceptable clutter in root.

## Dependencies to Add

```toml
# pyproject.toml [dependency-groups]
docs = [
    "mkdocs>=1.6",
    "mkdocs-material>=9.5",
    "mkdocs-api-autonav>=0.2",
]
```

## Commands Reference

```bash
# Install docs dependencies
uv sync --group docs

# Local development server (hot reload)
uv run mkdocs serve

# Build static site
uv run mkdocs build

# Deploy to GitHub Pages (from local)
uv run mkdocs gh-deploy
```

## Proposed `docs/` Structure

```
docs/
├── index.md              # Landing page
├── getting-started.md    # Quick start guide
├── guides/
│   ├── pre_commit.md     # Migrated
│   └── uv.md             # Migrated
├── api/                  # Auto-generated by mkdocs-api-autonav
│   └── (generated)
└── contributing.md       # Contribution guidelines
```

## Notes

- `mkdocs-api-autonav` requires docstrings in Google or NumPy style for best results.
- Material theme supports dark mode, search, and many UI features out of the box.
- GitHub Pages deployment can be automated via `mkdocs gh-deploy` or GitHub Actions.
